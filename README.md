AI Recipe Assistant (WPF)
Wikipedia および Wikibooks の API を活用し、料理名から自動的に「材料」と「作り方」を抽出・整形して管理できる C# / WPF アプリケーションです。

🌟 主な特徴
2段階ハイブリッド検索: レシピ専門の「Wikibooks」を優先し、見つからない場合は百科事典「Wikipedia」から情報を補完するフォールバック機能を搭載しています。

高度な構造化抽出: Wiki特有のマークアップ（==見出し== や *リスト）を解析し、読みやすい「・ リスト形式」に自動変換します。

インプレイス編集: 取得したレシピ情報をその場で自由に編集し、納得のいく内容で保存可能です。

ナレッジ蓄積: 保存したレシピはサイドバーの履歴から瞬時に呼び出せます。

モダン UI: ダークサイドバーとカード型デザインを採用し、視認性と操作性を両立しました。

🛠 技術スタック
Framework: .NET 6.0+ / WPF

Language: C# 10.0+

通信: HttpClient (非同期 I/O)

データ解析: System.Text.Json (DOM ベースの解析)

アーキテクチャ: サービス分離（RecipeService）によるクリーンな設計

📂 ファイル構成
MainWindow.xaml: DropShadow や角丸を駆使した直感的な UI 設計。

MainWindow.xaml.cs: UI 制御およびイベントハンドリング。

RecipeService.cs: API 通信、セクション抽出、正規化ロジックを担当。

SavedRecipes/: ユーザーが保存したレシピ（.txt）が格納されるディレクトリ。

🚀 使い方
検索: 画面上部の入力欄に料理名（例：「肉じゃが」「カルボナーラ」）を入力し、「Wikiで検索」をクリックします。

編集: 取得された「材料」や「手順」に自分なりのアレンジ（隠し味など）を書き加えます。

保存: 「保存」ボタンを押すと、タイトルをファイル名としてローカルに保存されます。

閲覧: 左側の履歴リストから、いつでも過去のレシピを確認できます。

🔧 カスタマイズ（エンジニア向け）
セクション抽出の拡張: ExtractSection メソッドの引数にキーワードを追加することで、栄養価や歴史などのセクションも抽出可能です。

フォント調整: txtResult の FontFamily を変更することで、お好みのエディタ環境に調整できます。

開発のこだわり（ロジスティクス思想）
このアプリは「情報の供給ライン」を重視しています。未加工の Wiki データをそのまま出すのではなく、システム内で「検品・加工（パース）」を行うことで、ユーザーがすぐに「調理（活用）」できる状態に整えて提供することを目的として設計されました。
